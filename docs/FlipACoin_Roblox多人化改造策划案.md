# Flip A Coin Roblox 多人化改造策划案

## 1. 项目定位

- 项目目标：把 Steam《Unfair Flips》的“被概率折磨但又停不下来”的体验，改造成适合 Roblox 的 8 人同桌、多人围观、轻社交、轻数值成长游戏。
- 核心卖点：不是单纯把单机抛硬币搬到 Roblox，而是把“别人也在受苦，而且你能现场看见”做成主要乐趣。
- 目标用户：
  - 喜欢轻玩法、强反馈、低门槛的 Roblox 大众玩家
  - 喜欢围观、起哄、刷高光瞬间的社交型玩家
  - 喜欢 rebirth、成长、收集硬币的轻数值玩家
- 推荐服务器规模：`8 人`
- 推荐单局体验：
  - 新玩家单次停留：`8-15 分钟`
  - 熟练玩家单次停留：`12-25 分钟`
  - 单次成功打出 10 连正面的平均目标：首日总游戏时长 `20-35 分钟` 内至少出现一次

## 2. 外部参考结论

### 2.1 Steam《Unfair Flips》的核心价值

- 起点极低：初始正面概率 `20%`
- 终局目标极简：达成 `10 连正面`
- 成长结构清晰：主要围绕
  - 硬币价值
  - 连击收益
  - 抛硬币速度
  - 正面概率
- 关键气质：它不是“公平成长”，而是“你越来越强，但命运还是会羞辱你”

结论：本作必须保留“目标简单、情绪强烈、永不保底”的灵魂，不适合一开始就堆太多 simulator 系统。

### 2.2 Roblox《Unfair Flips》的启示

- 已经验证了 Roblox 玩家愿意接受这个题材
- 该版本把起始概率提高到了 `25%`
- 加入了 gem、rebirth、reward、card 等 Roblox 常见长线系统
- Rolimon's 页面显示：
  - Max Players：`8`
  - Average Playtime：约 `5.85 分钟`
  - Visits：约 `4.2M`

结论：Roblox 版本确实需要长线成长，但如果系统太多，容易从“折磨人的硬币游戏”变成“普通挂机数值壳子”。

### 2.3 Roblox《Just Flip Bro》的启示

- 核心乐趣更偏“多人同场 + 看别人连续猜中”
- 平均游玩时长较短，传播性比深度更强
- Rolimon's 页面显示：
  - Max Players：`12`
  - Average Playtime：约 `1.05 分钟`
  - Visits：约 `2.7M`

结论：多人同场本身就能制造传播，但如果没有长线成长，内容消耗会非常快。

### 2.4 Roblox《Flip A Coin!》的启示

- 题材已被证明适合 Roblox 的轻数值表达
- 玩家接受“不同硬币 + 升级 + rebirth + world”这一套
- 但这类结构很容易滑向标准 simulator，削弱题材辨识度

结论：本作可以加入“不同硬币”和“rebirth”，但不建议首发就做多世界和重挂机化。

## 3. 本作设计结论

### 3.1 核心方向

- 保留 Steam 原作的四条成长线
- 强化 Roblox 特有的“同桌围观感”
- 用轻量 meta 维持留存，不把游戏做成大而空的 simulator
- 让多人存在感体现在“共视、起哄、共享高光”，而不是互相恶心

### 3.2 三条设计原则

1. 折磨感不能丢  
即使成长后，10 连正面也绝不能变成保底事件。

2. 社交感必须来自同桌现场  
玩家要持续看到别人正在失败、连中、上头、崩溃。

3. 长线成长要轻，不要脏  
首发先做清晰的成长环，不要上来就堆 5 种货币、20 个界面、3 个世界。

## 4. 最终产品形态

### 4.1 世界结构

- 一个主场景：`圆桌大厅`
- 核心物件：`1 张可坐 8 人的大圆桌`
- 每个座位对应 1 个玩家的独立抛币回合
- 玩家默认进入大厅，点击空位坐下开始游戏
- 旁观玩家可以站在桌后看，也可以排队等座
- AFK 超过 `90 秒` 自动离座

### 4.2 镜头与表现

- 坐下后默认进入轻俯视、偏近景镜头
- 中央看到自己硬币，周围能看到其他玩家的硬币与表现
- 别人的关键信息用轻量 Billboard 显示：
  - 玩家名
  - 当前连正面数
  - 当前装备硬币
  - 最近一次高光状态

### 4.3 核心操作

- PC：`Space` 或鼠标点击抛硬币
- Mobile：大按钮点击
- 长按支持连续输入
- 自动抛币不是默认白送，而是中期解锁的便捷功能

## 5. 核心玩法循环

### 5.1 单次循环

1. 抛硬币
2. 若为正面，获得现金并提升当前 streak
3. 若为反面，streak 归零
4. 用现金购买四类升级
5. 冲击 10 连正面
6. 达到合适节点后 rebirth，获得永久成长

### 5.2 一局中的情绪节奏

- 前期：极弱、抠门、容易连炸
- 中期：开始出现 4-7 连正面的“差一点”情绪
- 后期：概率与速度都上来，桌上会频繁出现“快成了”的围观氛围
- 终局：10 连正面达成时，必须有全桌可见的高光播报

## 6. 多人化改造重点

### 6.1 不是 PvP，而是同桌共视

本作不建议首发做互相干扰、偷 buff、打断 streak 的负向交互。原因：

- 被人搞输会让“概率导致的挫败”变成“社交导致的恶心”
- Roblox 公服环境下，负向交互很容易转化成骚扰和刷屏
- 原作的情绪来源是命运，而不是其他玩家

因此，本作的多人交互应该是：

- 看见别人抛
- 看见别人连中
- 为别人起哄
- 和别人共享桌面节奏
- 分享高光和翻车瞬间

### 6.2 首发社交系统

#### A. 热门 streak 播报

- 玩家达到 `3/5/7/9/10` 连正面时，全桌播报
- 视觉逐级增强：
  - `3 连`：轻提示
  - `5 连`：桌边灯光亮起
  - `7 连`：音效 + 屏幕边缘高亮
  - `9 连`：全桌聚光 + 所有人 HUD 提示
  - `10 连`：全桌庆祝动画 + 横幅播报

#### B. Table Hype 共享热度条

- 桌上所有玩家共用一个 `Hype` 条
- 任意玩家每次抛币都会给 Hype 加值：
  - 反面：`+2`
  - 正面：`+4`
  - 达成 `5/7/10` 连正面时额外 `+10/+15/+25`
- Hype 达到 `120` 时触发一次 `Table Fever`
- Table Fever 持续 `20 秒`
- Table Fever 效果：
  - 全桌现金收益 `+10%`
  - 全桌抛币间隔 `-10%`
  - 桌面特效升级
- 触发后 Hype 重置为 `0`

这个系统的作用是把“别人抛币”变成对你也有价值的现场氛围，而不是单纯背景动画。

#### C. Cheer 表情系统

- 每位玩家有 6 个基础桌面表情：
  - Cheer
  - Pray
  - Clap
  - Groan
  - Flex
  - Salt
- 表情不直接给数值，避免多开刷收益
- 作用是强化同桌社交氛围和短视频传播素材

### 6.3 不建议首发加入的多人系统

- 不建议首发做玩家互相下 debuff
- 不建议首发做真钱式对赌
- 不建议首发做可转移核心收益的送礼

这些系统可以作为后续版本尝试，但不适合首发定调。

## 7. 核心数值设计

## 7.1 基础规则

- 初始正面概率：`20%`
- 单次目标：`10 连正面`
- 初始单次抛币间隔：`1.80 秒`
- 初始正面基础收益：`5 Cash`
- 反面收益：`0`
- 正面时连击数 `+1`
- 反面时当前连击数重置为 `0`
- 概率全局硬上限：`60%`

保留 `60%` 上限很重要。即便到上限，10 连正面依然不是保底，戏剧性仍在。

### 7.2 单次正面收益公式

`HeadReward = 5 * ValueMultiplier * ComboMultiplier * FeverMultiplier * CoinPassiveMultiplier`

其中：

- `ValueMultiplier = 1.22 ^ ValueLevel`
- `ComboMultiplier = 1 + (CurrentStreak - 1) * ComboStep`
- `ComboStep = 0.20 + 0.05 * ComboLevel`
- `FeverMultiplier = 1.10`，仅在 Table Fever 期间生效

说明：

- `CurrentStreak` 指这次正面结算后的连击数
- 例子：ComboLevel = 4 时，ComboStep = `0.40`
- 如果当前是第 5 次连续正面，则 ComboMultiplier = `1 + 4 * 0.40 = 2.60`

### 7.3 四类核心升级

#### A. Value Upgrade

- 作用：提高每次正面的现金价值
- 等级上限：`20`
- 效果：`ValueMultiplier = 1.22 ^ Level`
- 升级价格公式：`Cost = round(10 * 1.55 ^ Level)`

关键节点：

| 等级 | 价格 | 乘区 |
| --- | ---: | ---: |
| 0 -> 1 | 10 | 1.22x |
| 1 -> 2 | 16 | 1.49x |
| 2 -> 3 | 24 | 1.82x |
| 4 -> 5 | 58 | 2.70x |
| 9 -> 10 | 510 | 7.30x |
| 14 -> 15 | 4480 | 19.70x |
| 19 -> 20 | 39292 | 53.10x |

#### B. Combo Upgrade

- 作用：提高连续正面的收益斜率
- 等级上限：`15`
- 效果：`ComboStep = 0.20 + 0.05 * Level`
- 升级价格公式：`Cost = round(20 * 1.60 ^ Level)`

关键节点：

| 等级 | 价格 | ComboStep |
| --- | ---: | ---: |
| 0 -> 1 | 20 | 0.25 |
| 1 -> 2 | 32 | 0.30 |
| 2 -> 3 | 51 | 0.35 |
| 4 -> 5 | 131 | 0.45 |
| 9 -> 10 | 1374 | 0.70 |
| 14 -> 15 | 14412 | 0.95 |

说明：  
当 ComboLevel = `15` 时，若打到 10 连正面，则最后一次正面收益乘区为 `1 + 9 * 0.95 = 9.55x`。

#### C. Speed Upgrade

- 作用：缩短抛币间隔
- 等级上限：`12`
- 效果：`FlipInterval = max(0.75, 1.80 * 0.93 ^ Level)`
- 升级价格公式：`Cost = round(25 * 1.65 ^ Level)`

关键节点：

| 等级 | 价格 | 抛币间隔 |
| --- | ---: | ---: |
| 0 -> 1 | 25 | 1.67s |
| 1 -> 2 | 41 | 1.56s |
| 2 -> 3 | 68 | 1.45s |
| 4 -> 5 | 185 | 1.25s |
| 7 -> 8 | 842 | 1.01s |
| 10 -> 11 | 3824 | 0.84s |
| 11 -> 12 | 6309 | 0.78s |

#### D. Bias Upgrade

- 作用：提高正面概率
- 等级上限：`16`
- 效果：每级 `+2.5%` 正面概率
- 升级价格公式：`Cost = round(40 * 1.70 ^ Level)`
- 概率结算：`FinalHeadsChance = min(60%, BaseChance + 2.5% * BiasLevel + CoinChanceBonus + PrestigeStartBias)`

关键节点：

| 等级 | 价格 | 理论概率增量 |
| --- | ---: | ---: |
| 0 -> 1 | 40 | +2.5% |
| 1 -> 2 | 68 | +5.0% |
| 2 -> 3 | 116 | +7.5% |
| 4 -> 5 | 334 | +12.5% |
| 7 -> 8 | 1641 | +20.0% |
| 11 -> 12 | 13675 | +30.0% |
| 15 -> 16 | 114132 | +40.0% |

### 7.4 新手节奏目标

- 第一次购买升级时间：`20-40 秒`
- 第一次出现 3 连正面：前 `2-4 分钟`
- 第一次达到 5 连正面：前 `5-10 分钟`
- 首次达成 10 连正面：
  - 中位数目标：`20-35 分钟` 总游戏时长
  - 不要求单 run 完成

### 7.5 新手引导任务

为了避免 20% 起点太干，新手前 5 分钟用任务送资源，而不是偷偷改概率。

| 任务 | 奖励 |
| --- | --- |
| 完成 10 次抛币 | 20 Cash |
| 购买 1 次升级 | 30 Cash |
| 达成 3 连正面 | 40 Cash |
| 触发 1 次 Table Fever | 1 个基础硬币皮肤 |

## 8. Rebirth 设计

### 8.1 为什么必须有 rebirth

- Roblox 玩家需要明确的长期成长
- 单局运气波动过大，必须给“虽然没通关，但这局也没白玩”的补偿
- rebirth 可以把运气挫败转化为长期推进感

### 8.2 命名建议

- UI 上直接叫：`Rebirth`
- 世界观包装名：`Mint Again`

### 8.3 Rebirth 解锁条件

满足任一条件可 rebirth：

- 本局最佳 streak 达到 `7`
- 本局总收益达到 `2500 Cash`
- 本局达成 `10 连正面`

这样既保留 10 连目标，也避免极端倒霉玩家完全卡死。

### 8.4 Rebirth 产物

- 永久货币名称：`Fate Shards`

### 8.5 Rebirth 收益公式

`FateShards = CashPart + StreakPart + FirstClearBonus`

其中：

- `CashPart = floor(log10(TotalCashEarned + 1) * 2)`
- `StreakPart` 由本局最佳 streak 决定
- `FirstClearBonus = 20`，仅首次达成 10 连正面时发放

#### Best Streak 对应的 StreakPart

| Best Streak | Shards |
| --- | ---: |
| 1-4 | 0 |
| 5 | 1 |
| 6 | 2 |
| 7 | 4 |
| 8 | 7 |
| 9 | 11 |
| 10 | 16 |

#### 例子

- 一局赚了 `3200 Cash`，最佳 streak `8`
- CashPart = `floor(log10(3201) * 2) = 7`
- StreakPart = `7`
- 本次 rebirth 奖励 = `14 Fate Shards`

### 8.6 Rebirth 的永久成长方式

Rebirth 不直接加面板大数值，而是给玩家“带着更好的开局再来一轮”。

#### 永久起始等级树

| 节点 | 效果 | 等级上限 | 单级价格 |
| --- | --- | ---: | --- |
| Polished Start | 开局额外获得 1 级 Value | 4 | 3 / 7 / 12 / 18 |
| Chain Start | 开局额外获得 1 级 Combo | 4 | 3 / 7 / 12 / 18 |
| Quick Start | 开局额外获得 1 级 Speed | 4 | 3 / 7 / 12 / 18 |
| Lucky Start | 开局额外获得 1 级 Bias | 4 | 4 / 8 / 14 / 20 |

说明：

- 起始等级只影响开局，不修改最终硬上限
- Lucky Start 本质上是“开局自带 Bias 等级”，仍受 `60%` 全局上限约束

### 8.7 Rebirth 的中期解锁内容

| 内容 | 解锁条件 |
| --- | --- |
| Auto Flip Toggle | 累计 rebirth `1` 次 |
| 第二枚可装备功能硬币 | 累计 rebirth `2` 次 |
| Fate Cards 系统 | 累计 rebirth `3` 次 |
| 特殊桌面皮肤 | 累计 rebirth `5` 次 |

## 9. 不同硬币系统

### 9.1 设计目标

- 给玩家长期收集目标
- 形成不同 build，但不破坏核心四升级的主体地位
- 不把硬币做成纯皮肤，也不做成一眼最强的数值碾压

### 9.2 首发硬币原则

- 首发 `6 枚` 功能硬币足够
- 每枚硬币只有 `1 个主被动`
- 不做复杂词条随机
- 不做装备栏堆叠

### 9.3 首发硬币清单

| 名称 | 获取方式 | 被动效果 |
| --- | --- | --- |
| Rusty Penny | 初始赠送 | 无被动，标准基准 coin |
| Lucky Nickel | 最佳 streak 达到 5 | 开局额外 `+2.5%` 正面概率 |
| Casino Token | 累计获得 5000 Cash | 正面收益 `+12%` |
| Quicksilver Coin | rebirth 1 次 | 抛币间隔 `-8%` |
| Echo Coin | 触发 10 次 Table Fever | `5 连正面` 以上时收益额外 `+15%` |
| Black Mint | 首次达成 10 连正面 | 开局额外 `+1` Combo 等级，但正面收益 `-8%` |

### 9.4 硬币平衡原则

- 单枚硬币提供的有效强度建议控制在 `5%-12%` 之间
- 功能硬币只改变起手风格，不取代核心 run upgrade
- 后续若要做稀有度，建议优先做视觉稀有度，不要立刻做战力稀有度

## 10. Auto Flip 设计

### 10.1 是否要做

要做，但不能从第一秒就让游戏变成纯挂机。

### 10.2 推荐方案

- 玩家 rebirth `1 次` 后解锁 `Auto Flip Toggle`
- Auto Flip 的速度上限与当前 Speed 等级绑定
- 手动输入保留轻微优势：
  - 手动：按当前 Speed 的完整速度
  - Auto Flip：按当前 Speed 的 `95%` 速度

这样 Auto Flip 是便捷功能，不是严格最优玩法。

### 10.3 付费设计建议

如果要卖 gamepass，不建议卖“只有付费才有自动抛币”。  
建议卖：

- 付费玩家开局直接可用 Auto Flip
- 或者 Auto Flip 的切换更方便
- 或者附带外观、音效、桌面特效

不要让免费玩家完全没有自动功能。

## 11. Fate Cards 设计建议

### 11.1 是否首发上线

不建议作为首发主系统。  
建议作为 `v1.1` 内容，在玩家完成 `3 次 rebirth` 后解锁。

原因：

- 首发核心应该先验证“8 人同桌 + 原作四升级 + rebirth”是否成立
- 卡牌会大幅提高系统复杂度和数值调试成本
- 如果首发就上卡牌，容易把核心乐趣稀释掉

### 11.2 推荐卡牌形态

- 每局最多持有 `2 张`
- 每 `50 次抛币` 提供一次 `3 选 1`
- 全是“本局临时增益”，不做长期收集卡池

### 11.3 卡牌样例

| 卡牌 | 效果 |
| --- | --- |
| Lucky Edge | 接下来 `20 次抛币` 正面概率 `+5%` |
| Greedy Palm | 接下来 `15 次正面` 收益 `+35%` |
| Quick Flick | 接下来 `30 次抛币` 间隔 `-20%` |
| Second Chance | 下次出现反面时，不会中断 streak，但只生效 1 次 |
| Fever Lover | 下一次 Table Fever 额外延长 `10 秒` |
| Soft Landing | streak 被打断时，返还本次理论正面收益的 `40%` |
| Echo Win | 达到 `7 连正面` 时立即获得 `1` Fate Shard，本局仅 1 次 |
| Final Push | 当 streak >= 8 时，正面收益再 `+25%` |

## 12. 任务与留存系统

### 12.1 Daily

每日给 `3` 个任务，保持目标简单：

| 任务 | 奖励 |
| --- | --- |
| 完成 150 次抛币 | 3 Fate Shards |
| 触发 3 次 Table Fever | 3 Fate Shards |
| 达成 6 连正面 1 次 | 4 Fate Shards |

### 12.2 Weekly

| 任务 | 奖励 |
| --- | --- |
| rebirth 3 次 | 稀有硬币皮肤 |
| 达成 8 连正面 2 次 | 10 Fate Shards |

### 12.3 Profile XP

- 正面：`+5 XP`
- 反面：`+2 XP`
- 达成 `5/7/10` streak 额外奖励：`+20 / +35 / +80 XP`
- 等级需求建议公式：`NeedXP = round(40 * Level ^ 1.35)`

Profile Level 奖励优先给：

- 表情
- 椅子/桌面皮肤
- 名牌边框
- 非战斗型硬币外观

## 16. 数据与埋点建议

首发必须跟的关键埋点：

- `seat_taken`
- `flip_result`
- `streak_peak`
- `upgrade_bought`
- `table_fever_triggered`
- `rebirth_done`
- `coin_equipped`
- `session_length`
- `leave_reason`

重点看这几个指标：

- 首日平均停留是否达到 `8 分钟+`
- 首局是否能在 `3 分钟内` 买到至少 2 次升级
- 新玩家首日 rebirth 率是否达到 `35%-45%`
- 服务器平均上座率是否达到 `4 人+`
- 达成首次 10 连正面的玩家比例是否达到 `20%-30%`

## 17. 一句话总结

这款游戏最正确的改造方式，不是把《Unfair Flips》做成普通 coin simulator，而是做成：

`一个 8 个人围坐在一起，亲眼看着彼此被命运折磨，又一起制造高光瞬间的多人抛硬币游戏。`

多人化要服务“现场感”，成长要服务“再来一局”，而不是让系统把题材本身淹没掉。
